# 設定頁共用規則

## 適用範圍

- 本檔適用於以設定物件為核心的 WinForms 設定頁。
- 本檔描述的是設定頁共用模式，不是單一頁面的專屬規則。

## 結構規則

- 設定頁外層 tab 控制項優先沿用既有 `hahahalib.ui.Tab_Controll_Base`。
- 設定容器內的設定物件以一個物件一個 tab 的方式呈現。
- tab 名稱、操作按鈕名稱優先使用中文。
- tab 內容以 `panel` 當底，不使用 `PropertyGrid`。

## 版面規則

- 欄位標題與輸入框維持同一行排列。
- 欄位標題字型維持 `16pt`。
- 下方固定保留底部操作區 panel，集中放 `儲存`、`取消` 按鈕。
- 若使用者已調整既有 layout，後續修改只做對齊，不要任意重排控制項座標。
- 若使用者已擺好控制項位置與尺寸，不要任意改按鈕大小、輸入框大小、或整組控制項的擺放位置。
- 若真的需要微調，僅限維持同一排的細部對齊，不得把控制項搬到另一組座標配置。

## 欄位規則

- 只有允許調整的設定欄位才提供輸入框。
- 標記為內建設定的欄位，不提供輸入框，維持由程式內建值控制。
- 若欄位屬於路徑型設定，可在輸入框後方保留 `...` 按鈕，用來開啟資料夾選擇視窗。

## 存讀流程

- `Form_Load` 時先設定頁面標題，再呼叫統一的 reload 方法回填畫面。
- reload 方法只回填目前允許編輯的欄位。
- save 方法只回寫目前允許編輯的欄位，再呼叫對應的 `Save_xxx()`。
- `取消` 按鈕流程為重新載入設定後再回填畫面。
- 若 reload 流程內有重新 `Load_All()`，可在 reload 後重新指定 `ha.Setting = hahaha.Setting_Box_`。
- save 流程若只是把畫面值回寫到 `hahaha.Setting_Box_` 再儲存，不要額外重設 `ha.Setting`。
